## Phase C: Backend Services & Auth Logic

### Step 7: Create auth_service.py

**Files:** `api/app/services/auth_service.py` — **NEW**

**Action:** Implement 6 functions per §13.1:

| Function | Description |
|----------|-------------|
| `create_access_token(user)` | JWT RS256, 15min, claims: sub, exp, iat, iss, plan |
| `create_refresh_token(user)` | JWT RS256, 7d, claims: sub, exp, iat, iss, type="refresh", ver |
| `verify_access_token(token)` | Decode + validate, require [sub, exp, iat, iss] |
| `verify_refresh_token(token)` | Decode + validate, require type="refresh" |
| `set_refresh_cookie(response, token)` | HttpOnly, SameSite=Lax, Secure=prod only |
| `clear_refresh_cookie(response)` | Delete cookie with matching attributes |

**Key implementation details:**
- Use `jwt.encode()` / `jwt.decode()` from PyJWT
- `algorithms=["RS256"]` mandatory on decode
- `issuer="tech-news-curator"` validation
- `settings.jwt_private_key` for signing, `settings.jwt_public_key` for verification

**Validation:**
- [ ] Token round-trip: encode → decode → same claims
- [ ] Expired token raises `jwt.ExpiredSignatureError`
- [ ] Invalid signature raises `jwt.InvalidSignatureError`

## Phase D: API Endpoints & Middleware

### Step 11: Create auth router

**Files:** `api/app/routers/auth.py` — **NEW**

**Action:** Implement 6 endpoints per §6.3:

| Endpoint | Handler | Rate Limit |
|----------|---------|-----------|
| `GET /auth/{provider}/login` | `oauth_login` | 10/min |
| `GET /auth/{provider}/callback` | `oauth_callback` | 10/min |
| `POST /auth/refresh` | `refresh_token` | 30/min |
| `POST /auth/logout` | `logout` | 10/min |

**Callback flow (§6.3.2):**
1. `authorize_access_token(request)` — token exchange
2. Extract user info (Google: `token["userinfo"]`, GitHub: GET /user + /user/emails)
3. Verify email (`email_verified` for Google, `verified && primary` for GitHub)
4. `upsert_user()` — create or link account
5. Generate JWT pair (access + refresh)
6. `set_refresh_cookie()` — httpOnly cookie
7. `RedirectResponse` → `AUTH_REDIRECT_URL`

**Validation:**
- [ ] `/auth/google/login` returns 302
- [ ] `/auth/unknown/login` returns 404
- [ ] `/auth/refresh` with valid cookie returns TokenResponse
- [ ] `/auth/refresh` without cookie returns 401
- [ ] `/auth/logout` clears cookie + increments token_version
